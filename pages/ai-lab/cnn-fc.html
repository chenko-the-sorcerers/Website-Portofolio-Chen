01

Peran Fully Connected Layer

Dari feature map ke keputusan klasifikasi — langkah terakhir CNN

Setelah beberapa layer Conv+ReLU+Pooling, CNN menghasilkan feature map berisi representasi abstrak dari gambar. Fully Connected (FC) layer bertugas menggabungkan semua representasi tersebut dan membuat keputusan final: gambar ini adalah kucing, anjing, atau pesawat?
🗺️
Input
Feature Maps
Tensor 3D — misal 7×7×512 setelah beberapa Conv+Pool. Menyimpan "di mana" dan "apa" fitur terdeteksi.
→
📊
Output
Skor Kelas
Vektor 1D dengan panjang = jumlah kelas. Misal 1000 angka untuk ImageNet. Softmax ubah menjadi probabilitas.
📐 Kalkulasi FC Layer
Dimensi output VGG-16 sebelum FC

Input gambar dan setelah Conv+Pool bertingkat
Input: 224×224×3 → setelah 5 MaxPool(2×2): 7×7×512
2
Flatten ke vektor 1D
7 × 7 × 512 = 25,088 neuron
3
FC Layer 1: 25,088 → 4,096 neuron
params = 25,088 × 4,096 + 4,096 = 102,764,544 ← 78% total params VGG!
4
FC Layer 2: 4,096 → 4,096, lalu → 1,000 (output)
Total FC params ≈ 119 juta dari total 138 juta VGG-16
02

Global Pooling vs Flatten

Arsitektur modern mengganti flatten + FC besar dengan Global Average Pooling

VGG menggunakan flatten → FC besar, tapi ini sangat boros parameter. Arsitektur modern seperti ResNet, MobileNet, dan EfficientNet menggunakan Global Average Pooling (GAP) — jauh lebih efisien.
Pendekatan Cara Kerja Params ke Classifier Contoh Model
Flatten + FC 7×7×512 → 25088 → 4096 → 1000 ~103 juta VGG-16
Global Avg Pool 7×7×512 → rata-rata → 512 → 1000 512,000 ResNet, MobileNet
GAP mengurangi params classifier 200× — dan sering menghasilkan akurasi lebih baik karena lebih sedikit overfit.
Global Average Pooling — Formula
GAP(Fk) = 1/(H×W) · ∑i,j Fk(i,j)
Fkfeature map channel ke-k
H×Wdimensi spasial feature map
Setiap channel feature map dikompresi menjadi satu angka — rata-rata aktivasinya.
03

Softmax & Distribusi Probabilitas

Mengubah skor mentah (logits) menjadi probabilitas yang menjumlah ke 1

Output layer terakhir CNN menghasilkan logits — vektor skor mentah, bisa negatif, positif, tidak perlu menjumlah ke 1. Softmax mengubah logits ini menjadi distribusi probabilitas yang proper.
Fungsi Softmax
P(yi) = ezi / ∑j ezj
zilogit untuk kelas ke-i
∑ ezjnormalisasi: jumlah semua exp logits

04

Cross-Entropy Loss

Fungsi loss untuk training klasifikasi — mengukur "seberapa salah" prediksi

Setelah Softmax menghasilkan probabilitas, kita butuh satu angka yang mengukur seberapa jauh prediksi dari label sebenarnya. Cross-entropy loss adalah pilihan standar untuk task klasifikasi.
Cross-Entropy Loss
L = −∑i yi · log(ŷi)
yilabel sebenarnya (one-hot: 1 untuk kelas benar, 0 untuk lainnya)
ŷiprobabilitas prediksi dari Softmax
Karena yi = 0 untuk semua kelas yang salah, hanya satu term yang tersisa: −log(P(kelas benar))

05

Arsitektur CNN Lengkap — Semuanya Bersama

Pipeline lengkap dari piksel input hingga keputusan klasifikasi

Sekarang kita sudah punya semua bahan: Konvolusi, ReLU, Pooling, FC Layer, dan Softmax. Inilah bagaimana semuanya bekerja bersama dalam pipeline klasifikasi gambar.
🏗️ Pipeline CNN Klasifikasi
Contoh: CIFAR-10 (32×32×3, 10 kelas)
1
Input
32×32×3 = 3072 nilai piksel
2
Conv2D(32 filters, 3×3) + ReLU + MaxPool(2×2)
32×32×3 → 16×16×32 params: 896 (Conv) + 0 (Pool)
3
Conv2D(64 filters, 3×3) + ReLU + MaxPool(2×2)
16×16×32 → 8×8×64 params: 18,496
4
Global Average Pooling
8×8×64 → 64 rata-rata per channel
5
FC Layer + Softmax
64 → 10 probabilitas params: 650
Total Parameter
896 + 18,496 + 640 + 10 (bias)
≈ 20,042 parameter
Jauh lebih kecil dari MLP setara — dan jauh lebih akurat!
