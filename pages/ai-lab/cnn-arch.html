01

Evolusi Arsitektur CNN

Dari AlexNet 2012 ke EfficientNet 2019 — 7 tahun lompatan akurasi dan efisiensi

Sejak AlexNet memenangkan ImageNet 2012 dengan top-5 error 15.3%, arsitektur CNN terus berevolusi. Setiap generasi memecahkan masalah spesifik yang menghambat generasi sebelumnya. Memahami mengapa setiap arsitektur dibuat adalah kunci memilih yang tepat.
2012
AlexNet
8 layers · 60M params
Top-5: 15.3% error
Pertama pakai ReLU + dropout + GPU training
2014
VGG-16
16 layers · 138M params
Top-5: 7.3% error
Konsisten pakai Conv 3×3. Simple tapi sangat besar
2015
ResNet-50
50 layers · 25M params
Top-5: 3.57% error
Skip connections — jaringan dalam akhirnya bisa ditraining
2017
MobileNetV2
53 layers · 3.4M params
Top-1: 72% acc
Depthwise separable conv — dirancang untuk mobile/edge
2019
EfficientNet-B7
813 layers · 66M params
Top-5: 1.8% error
Compound scaling — scale depth, width, resolution bersama
📉
Tren utama: Error turun dari 15.3% (AlexNet) ke 1.8% (EfficientNet) dalam 7 tahun, sementara jumlah parameter tidak selalu naik — ResNet-50 hanya butuh 25M params dibanding VGG-16 yang 138M, tapi jauh lebih akurat.
02

ResNet & Skip Connections

Satu ide sederhana yang membuka jalan untuk jaringan ratusan layer

Sebelum ResNet, menambah layer justru menurunkan akurasi — bukan karena overfitting, tapi karena vanishing gradient membuat layer awal tidak belajar. He et al. (2015) memecahkan ini dengan satu ide yang elegan: residual connection.
🔬 Diagram
Residual Block — Anatomy
Daripada belajar pemetaan H(x) langsung, ResNet belajar residual F(x) = H(x) − x, lalu tambahkan input asli: output = F(x) + x
📐 Matematika Residual
Kenapa skip connection memecahkan vanishing gradient?
1
Tanpa skip connection — gradient menyusut di setiap layer
// Backprop tanpa skip: kalikan Jacobian tiap layer
∂L/∂x0 = ∂L/∂xL · ∏l=1..L Wl ← bisa → 0 atau → ∞
2
Dengan skip connection — selalu ada jalur gradient langsung
xl+1 = F(xl, Wl) + xl ← shortcut langsung
3
Gradient selalu punya jalur "bebas hambatan" ke layer awal
∂xl+1/∂xl = ∂F/∂xl + 1 ← "+1" ini yang menyelamatkan gradient!
// Bahkan jika ∂F/∂x ≈ 0, gradient masih mengalir via "+1"
4
Hasil: ResNet-152 (152 layer) bisa ditraining — sebelumnya mustahil
ResNet-50: 25M params vs VGG-16: 138M params — lebih kecil, lebih akurat

03

EfficientNet & Compound Scaling

Satu formula untuk scale depth, width, dan resolusi secara bersamaan

Cara konvensional meningkatkan akurasi: tambah layer (depth), tambah channel (width), atau naikkan resolusi input. EfficientNet (Tan & Le, 2019) menunjukkan bahwa ketiganya harus di-scale secara proporsional dan bersamaan untuk hasil optimal.
Compound Scaling Formula
depth: d = αφ   ·   width: w = βφ   ·   resolution: r = γφ
α, β, γkonstanta yang dicari via grid search kecil (α=1.2, β=1.1, γ=1.15)
φcompound coefficient — satu angka yang mengontrol semua dimensi sekaligus
Constraint: α · β² · γ² ≈ 2, sehingga FLOPS naik ~2φ per step scaling

📐 Mengapa Compound Scaling Lebih Baik?
Scale satu dimensi vs compound scaling
1
Scale depth saja (tambah layer)
// Receptive field tidak naik karena resolusi sama
Akurasi naik sedikit lalu plateau — layer tambahan tidak berguna tanpa channel lebih
2
Scale resolution saja (input lebih besar)
// Butuh receptive field lebih besar untuk manfaatkan resolusi tinggi
Tanpa depth/width yang cukup, resolusi tinggi tidak terpakai optimal
3
Compound scaling — semua dimensi naik bersama
// Resolusi tinggi → perlu depth lebih untuk RF besar
// Depth lebih → perlu width lebih agar tidak bottleneck
EfficientNet-B7 vs ResNet-50: +8.1% akurasi dengan 8.4× lebih sedikit FLOPS

Pilih Backbone yang Tepat

Decision framework berdasarkan constraint nyata di production

Tidak ada satu backbone terbaik untuk semua kasus. Pilihan bergantung pada constraint deployment: apakah kamu butuh akurasi maksimal, latency rendah, model kecil untuk mobile, atau keseimbangan keduanya?
