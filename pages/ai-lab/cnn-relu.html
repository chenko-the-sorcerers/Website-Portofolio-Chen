01

Apa itu ReLU?

Fungsi aktivasi paling populer di deep learning — sederhana tapi powerful

Setelah operasi konvolusi menghasilkan feature map, nilainya bisa negatif maupun positif. Fungsi aktivasi menentukan apakah suatu neuron "aktif" atau tidak. ReLU (Rectified Linear Unit) adalah pilihan default hampir semua arsitektur CNN modern.
Definisi ReLU
ReLU(x) = max(0, x)
x < 0→ output = 0 (neuron "mati")
x ≥ 0→ output = x (pass-through linear)
Sesederhana itu. Potong semua nilai negatif jadi nol.

02

Mengapa Bukan Sigmoid?

Perbandingan matematis ReLU vs sigmoid — dan kenapa sigmoid kalah di deep network

Sebelum ReLU populer (sekitar 2012), sigmoid dan tanh adalah fungsi aktivasi standar. Keduanya punya masalah serius yang membuat training jaringan dalam menjadi sangat lambat atau gagal total.
Properti Sigmoid tanh ReLU
Formula 1 / (1+e−x) (ex−e−x) / (ex+e−x) max(0, x)
Output range (0, 1) (−1, 1) [0, ∞)
Komputasi Berat (exp) Berat (exp) Sangat ringan
Vanishing gradient Parah Parah Tidak (x>0)
Sparsity Tidak Tidak Ya (efisien)
⚡
Kecepatan komputasi. ReLU hanya butuh satu perbandingan (x > 0?). Sigmoid butuh operasi eksponen yang jauh lebih mahal. Dalam jaringan dengan jutaan aktivasi per forward pass, perbedaan ini sangat signifikan.
03

Vanishing Gradient Problem

Kenapa sigmoid "membunuh" gradient saat backpropagation

Saat training, gradient mengalir mundur dari output ke input melalui backpropagation. Di setiap layer, gradient dikalikan turunan fungsi aktivasi. Inilah masalahnya:
⚠️ Masalah Sigmoid
Gradient "mati" saat melewati sigmoid
1
Turunan sigmoid: σ'(x) = σ(x)(1 − σ(x))
// Nilai maksimum turunan sigmoid
σ'(0) = 0.5 × 0.5 = 0.25 ← maksimum!
σ'(2) = 0.88 × 0.12 = 0.105
σ'(5) ≈ 0.007 ← hampir nol
2
Gradient di layer ke-L dirantai (chain rule)
∂L/∂w1 = ∂L/∂wL × σ'L × σ'L−1 × ··· × σ'1
3
Dengan 10 layer sigmoid, gradient menyusut:
0.2510 = 0.0000009537 ← hampir nol, layer awal tidak belajar
4
ReLU: turunan = 1 jika x > 0
ReLU'(x) = 1 jika x > 0, 0 jika x ≤ 0
// Gradient tidak mengecil melewati neuron aktif!
🔑
Inilah alasan utama mengapa CNN dalam (VGG, ResNet) bisa ditraining dengan ReLU tapi gagal total dengan sigmoid. Gradient bisa mengalir dengan baik ke layer-layer awal, memungkinkan seluruh jaringan belajar.
04

Varian ReLU

ReLU punya satu kelemahan — dan beberapa varian untuk mengatasinya

ReLU punya masalah yang disebut "dying ReLU" — neuron yang sekali outputnya nol (karena semua input negatif), tidak bisa recover karena gradiennya juga nol. Beberapa varian hadir untuk mengatasi ini.
1
Leaky ReLU
Slope kecil (0.01) untuk nilai negatif, sehingga gradient tidak pernah benar-benar nol.
f(x) = x jika x > 0, 0.01x jika x ≤ 0
✓ Tidak ada dying neuron
2
Parametric ReLU
Slope negatif α dipelajari saat training — bukan hyperparameter tetap.
f(x) = x jika x > 0, αx jika x ≤ 0
α dioptimasi bersama weight lain
✓ Adaptif per layer/channel
3
GELU
Dipakai di Transformer (BERT, GPT). Versi smooth ReLU berbasis distribusi Gaussian.
f(x) = x · Φ(x)
Φ = CDF distribusi normal standar
✓ Default di model bahasa modern

05

Kapan Pakai Fungsi Aktivasi Apa?

Panduan praktis untuk memilih fungsi aktivasi

Pemilihan fungsi aktivasi tergantung pada tipe layer dan task. Ini panduan praktis berdasarkan penggunaan di arsitektur populer:
Posisi / Task Rekomendasi Alasan
Hidden layer CNN ReLU Cepat, tidak vanishing, sparse
Hidden layer dalam (ResNet, dll) ReLU / Leaky ReLU Cegah dying neuron di jaringan sangat dalam
Output klasifikasi biner Sigmoid Output harus di rentang (0,1) untuk probabilitas
Output klasifikasi multi-kelas Softmax Distribusi probabilitas yang menjumlah ke 1
Transformer / NLP GELU Dipakai di BERT, GPT — smooth gradient
💡
Default yang aman: Untuk sebagian besar CNN, mulai dengan ReLU. Jika kamu menemukan banyak "dead neurons" (aktivasi selalu nol), coba ganti ke Leaky ReLU. Jangan pakai sigmoid atau tanh di hidden layer kecuali ada alasan spesifik.
