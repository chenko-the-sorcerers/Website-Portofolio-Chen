Tokenization

The usual first step in any NLP pipeline. Chopping documents into smaller pieces (tokens) sounds simple, but edge cases abound — contractions, punctuation, emojis, compound words. We cover word-level, subword (BPE, WordPiece, SentencePiece), and character-level strategies and when to choose each.

BPEWordPieceSentencePieceNLTKspaCy 
from transformers import AutoTokenizer
tok = AutoTokenizer.from_pretrained(
  "bert-base-uncased")
ids = tok.encode(
  "Hello, world!", add_special_tokens=True)
# [101, 7592, 1010, 2088, 999, 102]
🧹

Basic Preprocessing

Depending on the goal, raw tokens need further cleaning. Case-folding collapses vocabulary size. Stop word removal strips high-frequency noise. Stemming aggressively chops suffixes (Porter, Snowball). Lemmatization does it properly using vocabulary and morphology — slower but smarter.

Case-foldingStop WordsStemmingLemmatizationNLTK
🏷️

Advanced Preprocessing — POS & NER

Tagging tokens with Part-of-Speech labels (NN, VBZ, JJ…) reveals grammatical roles. Named Entity Recognition (NER) extracts people, places, and organizations. Dependency parsing maps syntactic relationships. Constituency parsing shows phrase structure. Collectively, these unlock the deeper grammar of a sentence.

POS TaggingNERDependency ParsespaCyConstituency 
import spacy
nlp = spacy.load("en_core_web_sm")
doc = nlp("Apple is in Cupertino.")
for ent in doc.ents:
  print(ent.text, ent.label_)
# Apple ORG   Cupertino GPE

Text Representations

To run math on language, we first need to turn words into numbers — and the choice of representation matters enormously.

🛍️

Bag-of-Words & Document Similarity

The simplest possible representation — a vector of word counts, ignoring order. Despite its naivety, BoW still powers many real-world systems. We derive document similarity using cosine distance, build vocabulary matrices, and explore when this sparse representation is surprisingly effective.

BoWCountVectorizerCosine SimilaritySparse Matrix
📊

TF-IDF & Document Search

Term Frequency — Inverse Document Frequency fixes BoW's biggest flaw: common words dominate. TF-IDF down-weights words that appear in every document and up-weights rare discriminative terms. We build a simple but effective document search engine from scratch using TF-IDF vectors and cosine retrieval.

TF-IDFTfidfVectorizerDocument SearchRetrievalsklearn 
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

vec  = TfidfVectorizer()
X    = vec.fit_transform(corpus)
sims = cosine_similarity(query_vec, X)
🔢

Word Vectors & Embeddings

The breakthrough that changed NLP — vectors where similar words are geometrically close. Word2Vec (CBOW & Skip-gram) learns from co-occurrence. GloVe captures global statistics. FastText handles out-of-vocabulary words via subword embeddings. Downstream tasks can now leverage pre-trained semantic spaces as starting points.

Naïve Bayes Text Classification

Fast, interpretable, and surprisingly powerful for text. Naïve Bayes uses Bayes' theorem with the strong independence assumption that each word contributes independently to the class probability. We cover Bernoulli and Multinomial variants, Laplace smoothing for unseen words, and build a full sentiment classifier that rivals modern baselines on many benchmarks.

Topic Modelling — LDA

What do you do with a pile of unlabelled documents? Latent Dirichlet Allocation models each document as a mixture of topics, and each topic as a distribution over words. The model discovers hidden thematic structure without any labels. We cover the generative process, how to tune the number of topics, and build an LDA pipeline with Gensim for unsupervised document exploration.

Recurrent Neural Networks & Language Models

Recurrence adds memory — a hidden state carries information across time steps. Vanilla RNNs suffer from vanishing gradients over long sequences. LSTMs solve this with forget, input, and output gates. GRUs offer a lighter alternative. We build a character-level language model that generates text, and a POS tagger that processes sequences end-to-end.

RNNLSTMGRULanguage ModelVanishing GradientTeacher Forcing 
class LSTMClassifier(nn.Module):
  def __init__(self, vocab, emb, hid):
    super().__init__()
    self.emb  = nn.Embedding(vocab, emb)
    self.lstm = nn.LSTM(emb, hid,
                         batch_first=True)
    self.fc   = nn.Linear(hid, 2)

Sequence-to-Sequence & Attention

Many NLP tasks are sequence transformation problems — translation, summarization, question answering. The encoder-decoder architecture compresses the input into a context vector then decodes the output. The bottleneck hurts for long inputs. Bahdanau Attention fixes this by letting the decoder selectively attend to any encoder state. We build a neural machine translator and visualize the attention heatmap.

Transformers From Scratch

The 2017 "Attention is All You Need" architecture replaced recurrence entirely. Positional encoding injects token order without recurrence. Layer norm and residual connections stabilise training. The encoder stack (BERT-style) vs decoder stack (GPT-style) vs full encoder-decoder (T5). We build a mini-transformer from scratch in PyTorch and use it to answer questions.

Pre-training & Fine-tuning

Foundation models are pre-trained on massive corpora with self-supervised objectives — Masked Language Modelling (BERT), Causal LM (GPT), or denoising (T5). Fine-tuning adapts them to specific tasks with far less data. Parameter-efficient fine-tuning via LoRA and adapters makes it possible to specialise large models on consumer hardware.

RAG — Retrieval Augmented Generation

LLMs hallucinate when their parametric memory runs dry. RAG grounds the model in fresh, factual context by retrieving relevant chunks at inference time. Documents are split, embedded into a vector store (FAISS, Chroma, Pinecone), then the top-k nearest chunks are injected into the prompt before generation. Hybrid search (dense + sparse BM25) further improves recall.

Vector StoreFAISSBM25LangChainLlamaIndexHybrid Search 
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings

db  = Chroma.from_documents(
       docs, OpenAIEmbeddings())
ret = db.as_retriever(search_k=4)
chain = RetrievalQA.from_chain_type(
          llm, retriever=ret)

Evaluation in Modern NLP

Perplexity measures how surprised a language model is by held-out text — lower is better. BLEU and ROUGE score generated text against references. BERTScore leverages contextual embeddings for semantic similarity beyond n-gram overlap. For RAG systems, faithfulness and answer relevance require specialized evaluation frameworks like RAGAS.
